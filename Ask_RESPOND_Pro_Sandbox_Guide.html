DSL Consultation Tool — Demo Environment
⚛ Ask RESPOND Pro is a peer consultation tool designed for Designated Safeguarding Leads (DSLs), Deputy DSLs, and senior safeguarding professionals. This sandbox version gives you full access to the tool in a demo environment so you can evaluate it before committing to a full deployment.
⚠ Sandbox — not for live cases. This environment is for evaluation purposes only. Do not enter real names, school details, pupil information, or identifiable case data. Use fictional or anonymised scenarios.

What Ask RESPOND Pro Does

  • Supports complex decision-making — threshold analysis (S17 vs S47), LADO referral criteria, multi-agency coordination
  • Provides frameworks for thinking — not just answers, but structured approaches to weighing competing factors
  • Embeds statutory knowledge — KCSIE 2025, Working Together 2023, Children Acts, court order pathways
  • Offers contextual safeguarding analysis — 4Ps framework, extra-familial harm considerations, pattern recognition
  • Supports defensible decision-making — language and reasoning that withstands scrutiny in reviews or inspections
  • Respects your professional judgement — you remain accountable; the tool supports, not replaces, your expertise
⚙ Powered by RESPOND. Responses are grounded in the RESPOND Safeguarding Framework — a seven-step duty of care model (Recognise, Engage, Support, Pause, Offer, Notify, Document) — alongside the ACT theoretical foundation (Active Intervention, Contextual Safeguarding, Trauma-Informed Practice).

Getting Started

Accessing the Sandbox

  1. Navigate to the sandbox URL you have been given
  2. Enter your access code (provided by RESPOND Safeguarding)
  3. Tick the checkbox confirming you understand this is a demo environment
  4. Click Access Sandbox

Access codes are single-use tracking codes assigned per evaluator. Each code is tied to one recipient so usage can be monitored. Do not share your code.

Sessions last 4 hours from login. Closing the browser tab ends the session.

Effective Consultation

Ask RESPOND Pro works best when you give it context. Use fictional or anonymised scenarios that reflect the kinds of cases you actually deal with. The more specific the scenario, the more targeted the response.

Example Prompts to Try

  • "Help me think through whether this meets threshold for referral"
  • "I'm unsure if this is LADO or internal — can we work through it?"
  • "Multiple low-level concerns accumulating — when does it tip?"
  • "Information sharing dilemma — family refusing consent"
  • "Child-on-child case — how do I balance both children's needs?"
  • "Strategy meeting prep — what should I be ready for?"
  • "Parents are refusing to engage with early help. At what point does this become S47?"
  • "I need to justify why I'm not referring to MASH despite concerns. Help me document the rationale."

Priority Classification

Every response to a safeguarding concern opens with a colour-coded priority tag that signals the urgency level. This is reassessed on each turn — it does not carry forward automatically.

PriorityMeaning
REDImmediate action required. Imminent or serious risk. Cannot wait.
AMBERSignificant concern requiring action today. Not immediately life-threatening.
GREENStandard process. Manageable through normal channels without urgency.
BLUEPolicy, legal, or procedural query. No active concern identified.
GREYGeneral guidance, training, or administrative question.

What Ask RESPOND Pro Knows

Statutory Framework

  • KCSIE 2025 — DSL duties, Part 4 (allegations against staff), Part 5 (safer recruitment)
  • Working Together 2023 — multi-agency arrangements, threshold documents, information sharing
  • Children Act 1989 — S17, S20, S31, S43, S46, S47 duties and powers
  • Children Act 2004 — S10, S11, S14B–14D arrangements
  • Education Acts — S175/S157 safeguarding duties

Embedded Knowledge

  • Threshold decision framework (S17 vs S47 indicators)
  • LADO referral criteria (all four behaviour categories)
  • Court orders and pre-proceedings pathways
  • Child Protection Conference process and DSL role
  • Section 47 enquiry outcomes
  • Immediate protection powers (EPO, PPO)
  • 7 Golden Rules for information sharing
  • Contextual safeguarding (4Ps framework)
  • Professional optimism warning signs
  • RAG risk assessment indicators
ⓘ Note on citations. Ask RESPOND Pro can quote directly from KCSIE 2025 and Working Together 2023 where the relevant passage supports the response. Quotes are referenced by document and paragraph. Where no direct quote is available, the tool uses unquoted statutory framing.

Professional Boundaries

What Pro Does NOT Do

  • Make threshold decisions for you — it supports your thinking; you decide
  • Give definitive legal advice — for specific legal questions, consult your legal team
  • Replace multi-agency consultation — use it alongside, not instead of, professional discussions
  • Guarantee outcomes — safeguarding involves uncertainty; the tool helps you manage that professionally
  • Know your local protocols — it provides national guidance; you apply local context
  • Tell you who your DSL is — the tool is for DSLs, not a route to them
⚖ Accountability: You remain accountable for all safeguarding decisions. Ask RESPOND Pro is a thinking partner, not a decision-maker. Document your reasoning, not just the tool's output.

Privacy & Security in the Sandbox

  • Do not enter real case information — this is an evaluation environment, not a live system
  • No permanent storage — conversations are not logged or retained; closing the browser clears the session
  • Sessions expire after 4 hours — for security, sessions are tab-scoped and close with the browser
  • Access codes are tracked — usage is monitored per code to support the evaluation process
  • Full deployment includes additional controls — the production version can include school-specific configurations, local contacts, and case management system integration

Evaluating the Sandbox

Things Worth Testing

  • Run a scenario you have dealt with recently (anonymised) — does the response reflect how you would think it through?
  • Ask a threshold question where the answer is genuinely uncertain — does the tool handle ambiguity professionally?
  • Test the LADO gate — raise a concern involving only children and check it doesn't mention LADO
  • Ask a non-safeguarding question — does it redirect appropriately without lecturing?
  • Try the voice input if you typically work hands-free
  • Use the print function — does the output work as a record you could file?

Questions to Consider

  • Does it sound like an experienced DSL, or like a generic AI?
  • Does it support your judgement rather than replacing it?
  • Is the priority classification (RED/AMBER/GREEN/BLUE/GREY) calibrated appropriately?
  • Would your team use this? What would they need from a full deployment?

Troubleshooting

"My access code isn't working"

  • Check the code is entered exactly as provided — the field auto-uppercases but watch for typos
  • Codes are case-insensitive and do not expire, but are specific to this sandbox deployment
  • Contact darren@respondsafeguarding.org if the code continues to fail

"The response is too slow"

  • Complex scenarios take a few seconds — this is normal; the model is reasoning through the question
  • If the request times out, try again — the sandbox uses the same infrastructure as the production tool

"The response is too cautious / non-committal"

  • Pro does not make decisions for you — this is intentional, not a limitation
  • Ask for frameworks, not just answers: "How should I think about this?" rather than "What should I do?"
  • Provide more context about the complexity you're navigating

"The voice input isn't working"

  • Voice input requires a secure (HTTPS) connection and microphone permission in the browser
  • If the mic button is greyed out, check your browser's microphone settings
