2. What This Sandbox Does and Does Not Collect

✓ This sandbox does not log, store, or transmit any conversation content. Nothing you type into the chat is retained after your session ends. There is no server-side logging of queries or responses.

What happens when you use the tool

  • Your typed input is sent in real time to the Anthropic API (the AI model provider) to generate a response. It is not stored by us.
  • Anthropic processes your input in accordance with their own terms of service and privacy policy. We have configured the API in a way that does not involve persistent data retention on their side, consistent with standard API usage.
  • Your session is held in your browser's sessionStorage only. This means it exists solely within your browser tab and is deleted automatically when you close it. We cannot access it.
  • Your accessibility preferences (text size, contrast, etc.) are stored in your browser's localStorage on your own device only. We cannot access these.

Document uploads

The sandbox includes an optional document upload facility. Where you choose to upload a document, the following applies:

  • Processing is entirely client-side. Text is extracted from your document within your browser using JavaScript libraries (PDF.js for PDF files, Mammoth for Word documents). The document file itself is never transmitted to the application server or to Netlify.
  • Extracted text is held in browser memory for the session. The document text remains available in your browser tab so it can be referenced across multiple conversation turns. It exists only in JavaScript memory — not in localStorage, sessionStorage, or any server-side storage. When the tab closes, it is gone.
  • The document is included in every API request for the session. Each message you send while a document is loaded will include the document text as context. This allows the AI to refer to it consistently across the conversation. The document is not stored by the application at any point.
  • Documents are truncated at 12,000 characters to stay within API context limits.
⚠ Do not upload real documents. This is a demo environment. Do not upload any document containing real names, pupil information, staff details, medical information, or any other identifiable or confidential data. Use anonymised or fictional documents only.

What the access code system records

Access codes are verified server-side at login. The verification process checks whether the code you enter matches a configured value. No record of which code was used, when, or by whom is written to any log or database. The codes exist solely to control access to the sandbox.

What we do not collect

  • No names, email addresses, or contact details (unless you choose to contact us directly)
  • No IP addresses or device identifiers
  • No conversation content or case information
  • No cookies beyond those set by your browser for session management
  • No analytics or tracking scripts

3. The AI Model Provider

Ask RESPOND Pro uses the Claude API provided by Anthropic, PBC. When you send a message, it is transmitted to Anthropic's servers to generate a response. Anthropic is a sub-processor in this context.

Anthropic's privacy policy is available at anthropic.com/privacy. We have configured our API usage under standard commercial API terms, which do not permit Anthropic to use inputs to train their models without separate consent.

⚠ Important: This is a demonstration environment. You must not enter real names, identifiable pupil information, staff details, or live case data into this tool. Any information you choose to enter is processed by Anthropic in accordance with their privacy policy. Enter only anonymised or fictional scenarios.

4. Legal Basis for Processing

To the extent any personal data is processed in connection with this sandbox (for example, if you contact us directly with feedback), the legal basis is legitimate interests (UK GDPR Article 6(1)(f)) — specifically, our legitimate interest in evaluating and improving safeguarding technology for use in educational settings.

Because this sandbox does not collect or process personal data through normal use, UK GDPR obligations are engaged only in the limited circumstances described above.

6. Your Rights

Under UK GDPR, you have the following rights in relation to any personal data we hold about you:

  • Right of access — to request a copy of your personal data
  • Right to rectification — to request correction of inaccurate data
  • Right to erasure — to request deletion of your data where there is no lawful basis to retain it
  • Right to object — to object to processing based on legitimate interests
  • Right to data portability — to receive your data in a portable format
  • Right to lodge a complaint — with the Information Commissioner's Office (ICO)

Given the minimal data processed by this sandbox, the practical exercise of most of these rights will not apply. To exercise any right, or to raise a concern, contact us at darren@respondsafeguarding.org.

You may also contact the ICO: ico.org.uk  |  0303 123 1113

7. Security

The sandbox is hosted on Netlify's infrastructure, which provides HTTPS encryption in transit for all data. Access is controlled by individual access codes distributed to named evaluators only. No conversation data is written to disk or transmitted beyond the Anthropic API call.

Despite these measures, no system is entirely without risk. You remain responsible for the information you choose to enter and should not input real case information under any circumstances.
